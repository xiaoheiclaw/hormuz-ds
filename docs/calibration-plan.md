## 1. 校准目标

系统有三类需要校准的参数，校准目标不同：

| 类别 | 参数 | 目标 | 方法 |

## 2. ACH 校准

### 2.1 现状

```yaml
# m1_ach.py 当前 LR 映射
LR 范围: {0.77, 0.95, 1.0, 1.05, 1.3}
先验: H1=47.5%, H2=47.5% (H3 挂起，+5% 各分)
log-odds cap: ±3.0 (95% 上限)
```

### 2.2 问题

- LR 范围是设计时拍脑袋定的，没有用真实数据验证
- 不知道哪个观测对后验影响最大（敏感性未测）
- 无基准答案：什么情况下 H1=70% 是"对的"？

### 2.3 校准方案

**方案 A：情景回测（推荐）**

用已知结局的历史事件，喂入模拟观测，看后验是否收敛到正确方向：

| 情景 | 模拟输入 | 期望后验 |
|------|---------|---------|
| 2019 油轮袭击（6 月） | O01=0.3, O03=0.5, O04=0.3 | H2 > H1（IRGC 有意为之，能力未衰） |
| 2019 袭击停止（9 月） | O01→0.05, O02→0.8 | H1 上升（但实际是政治停火，测试灵敏度） |
| 假设：Day30 全面衰竭 | O01→0, O03→0, O04→0, O05→0 | H1 >> H2（>85%） |
| 假设：Day30 持续封锁 | O01=0.5, O03=0.7, O04=0.5, O05=0.7 | H2 >> H1（>70%） |
| 当前实际（Day14） | 真实 LLM 提取值 | H2 主导（已验证：74%） |

**执行**：写 `tests/calibration/test_ach_scenarios.py`，每个情景一个 test case，断言后验方向正确。不断言精确数值，只断言方向和大致范围。

**方案 B：敏感性分析**

逐个观测变化 ±0.3，测量后验变化幅度：

```
对每个 Oi:
  baseline = run_ach(all_obs)
  high = run_ach(all_obs with Oi += 0.3)
  low = run_ach(all_obs with Oi -= 0.3)
  sensitivity[Oi] = |high.h1 - low.h1|
```

输出敏感性排序，确认强判别力观测（O02-O05）确实影响最大。

## 3. 物理参数校准

### 3.1 锚点

| 参数 | 当前值 | 校准锚 | 验证方法 |
|------|--------|--------|---------|
| effective_disruption_rate（有效中断率） | 0.80 | R02: 2026-03 实际 13/~60 船 ≈ 78% 通行中断 | 与 R02 基本一致，暂维持 |
| irgc_decay_mean_weeks | 6.0 | 无直接锚点 | 敏感性分析：4/6/8 周对 T1 影响 |
| sea_mines_range | [20, 100] | R03: 1991 海湾战争 ~1300 枚 | 当前范围偏保守，可能需扩大 |
| convoy_start_mean_weeks | 5.0 | R03: 1991 扫雷 ~6 周 | 基本合理 |
| pipeline_max_mbd | 4.0 | 延布+ADCOP 公开产能 | 不变 |
| spr_rate_mean_mbd | 2.5 | DOE 历史释放记录 | 2022 SPR 释放峰值 ~1.0 mbd，2.5 偏高 |
| buffer_ramp d14_plus_mbd | 7.0 | 管道+SPR+绕行理论极限 | 需验证各子项是否可加 |

### 3.2 执行

1. **effective_disruption_rate（有效中断率）**：正常流量中被中断的比例。20.1 × 0.80 = 16.0 mbd 缺口。R02（13/60 船 ≈ 78% 中断）基本吻合，暂维持
2. **spr_rate_mean_mbd**：已从 2.5 调至 2.0。2022 年美国 SPR 实际释放 ~1.0 mbd（180M bbl / 6 months），2.0 介于理论极限和历史实际之间
3. **sea_mines_range**：上限 100 可能偏低，1991 海湾伊拉克布 ~1300 枚，但伊朗库存和布设能力不同

## 4. LLM 提取校准

### 4.1 现状

- 动态 context（conflict_day + previous_obs）已注入
- 二级关键词过滤（STRONG/WEAK）已启用
- 文章全文（HTML→text, 3000 字截断）已启用

### 4.2 校准方案

**人工标注对比**：

1. 选 10 篇典型文章（覆盖不同来源和话题）
2. DT 人工标注期望观测值（O01-O14 + 信号）
3. 跑 LLM 提取，对比人工标注
4. 计算偏差：哪些观测 LLM 系统性偏高/偏低
5. 调整 prompt 中的 calibration 语句

**关键词过滤验证**：

1. 随机抽 20 篇被过滤掉的文章
2. 人工判断是否有漏网的相关文章
3. 扩充 STRONG/WEAK 词表

### 4.3 已知偏差

| 偏差 | 原因 | 修复方向 |
|------|------|---------|
| O01 在无明确攻击数据时偏低 | LLM 保守，无数字则不敢给高值 | prompt 加"active conflict baseline" |
| B 组观测（O07-O11）经常缺失 | 新闻很少报道保费、运费精确数字 | 依赖手动录入，LLM 作为 fallback |
| 信号提取偏多 | LLM 倾向于"发现"信号 | 已有 valid_keys 白名单过滤 |
